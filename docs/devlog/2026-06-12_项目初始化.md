## Qt 6.11 兼容性问题

### 1. Qt Connectivity 改名
Qt 6.8+ 将 `Qt Bluetooth` 和 `Qt NFC` 合并为 **Qt Connectivity** 元模块。
- Maintenance Tool 中勾选 `Qt Connectivity` 而不是 `Qt Bluetooth`
- CMake 中仍用 `find_package(Qt6 REQUIRED COMPONENTS Bluetooth)` ——不受影响

### 2. QLowEnergyController::createCentral API 变化
Qt 6.11 中 `createCentral()` 不再接受 `QBluetoothAddress`，只接受 `QBluetoothDeviceInfo`。
- 旧写法：`QLowEnergyController::createCentral(QBluetoothAddress(addr), this)`
- 新写法：先构造 `QBluetoothDeviceInfo(QBluetoothAddress(addr), "", 0)` 再传入
- BleConnection.cpp 中已适配

### 3. enum class State 模板推导失败
`connect(connection, &BleConnection::stateChanged(State), ...)` 在 Qt 6.11 + MSVC 下无法匹配 lambda/slot。
- 解决：将信号参数从 `State` 改为 `int`，emit 时 `static_cast<int>(state)`
- 接收端手动 cast 回 `State`

### 4. BleDeviceInfo 需要持有 QBluetoothDeviceInfo
因为 createCentral 需要 `QBluetoothDeviceInfo`，而原来 BleScanner 只保存 name/address/rssi。
- 解决：BleDeviceInfo 增加 `qtInfo` 字段
